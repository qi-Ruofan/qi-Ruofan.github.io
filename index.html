javascriptjavascript——this指向
更新于 2020-09-06
javascript
this

this的设计目的

指向函数运行时所在的环境。

this的绑定规则：

  1. 默认绑定（严格/非严格模式）
  2. 隐式绑定
  3. 显式绑定
  4. new绑定
  5. ES6箭头函数绑定

默认绑定

  1. 默认绑定——严格模式
    严格模式下不能将全局对象window作为默认绑定，此时this会绑定到undefined,但是在严格模式下调用函数则不会影响默认绑定。
  2. 默认绑定——非严格模式
    在非严格模式下，此时this指向全局对象

隐式绑定

  • 当函数作为对象的属性存在时，通过对象属性执行函数时，此时隐式绑定规则会将this绑定到对象上
  • 通过赋值操作后执行函数，会应用默认绑定
  • 再回调函数中执行，也对丢失this绑定，应用默认绑定

显式绑定

（通过call，apply，bind绑定）
call：使用一个指定的this和若干个指定的参数调用某个函数或方法
call做了哪些事：

  • 将函数设为对象的属性
  • 指定函数的this，并进行传参
  • 执行&删除函数
  • 判定如果没有指定要绑定的this，非严格模式下默认指向全局对象
    可以看到值调用call方法后，会将this绑定到指定对象，所以称为显示绑定
    多次调用以第一次为准
    没有传入指定对象/null/undefined，采用默认绑定

new绑定

构造函数在使用new后，执行了什么操作

  • 他创建（构造）了一个全新的对象
  • 他会被执行[[Prototype]]也就是__proto__连接
  • 他使this指向新创建的对象
  • 通过new创建的每个对象将最终被[[Prototype]]链接到这个函数的prototype对象上
  • 如果函数没有返回对象类型Object（包含Function,Array,Date,RegExg,Error），那么new表达式中的函数调用将返回该对象引用
    所以在使用new调用构造函数后，会构造一个新对象并将函数调用中的this绑定到新对象上
    构造函数存在返回值，可以将函数的返回值分成三种情况：
  • 返回一个对象
  • 没有返回值，即默认返回undefined
  • 返回基本数据类型
    所以使用new绑定时，需要判断函数返回的值是否为一个对象，如果是对象，那么this会绑定到返回的对象上。

ES6箭头函数绑定

ES6新增了一种函数类型：箭头函数，箭头函数调用时无法使用上面四种规则了，它和普通函数最不同的一点就是对于箭头函数的this指向，是根据它外层（函数/全局）作用域来决定。
箭头函数this绑定后无法被修改

规则优先级

  1. new绑定
    var obj = new Foo();
    this绑定新的对象上
  2. 显示绑定
    var obj = foo.call(bar);
    this绑定到指定对象上
    若指定对象为null/undefined或着没传，则使用默认绑定规则
  3. 隐式绑定
    var obj = bar.foo();
    this绑定到调用方法的对象上
  4. 默认绑定
    foo();
    this在严格模式下绑定到undefined
    在非严格模式下绑定到全局对象

跨域请求
更新于 2020-09-06
浏览器
跨域

document.domain

  • 只能设置值为当前域或当前域的父域
  • 任何对document.domain的赋值操作，都会导致端口号被重写为 null。使用 document.domain 来允许子域安全访问其父域时，您需要在父域和子域中设置 document.domain 为相同的值。

嵌入跨源的资源

  • <script src="..."></script> 标签嵌入跨域脚本。语法错误信息只能被同源脚本中捕捉到。
  • <link rel="stylesheet" href="..."> 标签嵌入CSS。由于CSS的松散的语法规则，CSS的跨域需要一个设置正确的 HTTP 头部 Content-Type 。不同浏览器有不同的限制： IE, Firefox, Chrome, Safari (跳至CVE-2010-0051)部分 和 Opera。
  • 通过 <img> 展示的图片。支持的图片格式包括PNG,JPEG,GIF,BMP,SVG,…
  • 通过 <video> 和 <audio> 播放的多媒体资源。
  • 通过 <object>、 <embed> 和 <applet> 嵌入的插件。
  • 通过 @font-face 引入的字体。一些浏览器允许跨域字体（ cross-origin fonts），一些需要同源字体（same-origin fonts）。
  • 通过 <iframe> 载入的任何资源。站点可以使用 X-Frame-Options 消息头来阻止这种形式的跨域交互。

assert

最近读了《JavaScript忍者秘籍》，上面提到了assert函数，用来断言代码，确保代码的运行符合我们的预期，挺有意思，平时做一些小测试都用得上。

该函数的形式一般是assert(condition, message)

  • 第一个参数是一个条件，来判断输入
  • 第二个参数是在返回为false的时候，显示的一个消息

代码测试：

我们还可以用来测试代码性能：


<script>
  function assert (condition, message) {
    if (!condition) throw new Error(`${message}`)
  }

  let start = new Date().getTime();
  for(let i = 0 ; i < 10000; i++) {
   //输入你要测试的代码
  } 
  let end = new Date().getTime() - start;

  assert(false, "Measured time:" + end)
</script>

【javascript】作用域详解
更新于 2020-09-06
JavaScript
作用域

作用域分类

  • 全局作用域
  • 局部作用域/函数作用域
  • 块级作用域（es6以后）
  • 动态作用域（this）

全局作用域

特性：

  • 在函数外声明的全局变量在任何地方都能拿到，两个不同的文件也能取到
  • 没用var声明的变量虽然也具备全局作用域但是严格意义上不是全局变量（不可被删除），而是全局属性（可被删除）
  • 在函数中没用var声明的变量，具备全局作用域。

局部作用域（函数作用域）

特性：

  • 函数外部无法主动访问函数内部的信息
  • 共享方式：return/闭包

块级作用域

特性：

  • 行程块级作用域后，外部无法访问到块内部
  • 必须和let或const配合，行程暂时性死区

动态作用域

特性：

  • 可以通过bind,apply,call改变作用域指向

扩展：

let & const

  • let和const声明的变量不允许重复定义
  • let和const声明的变量不具备变量提升
  • let和const具有块级作用域
  • let和const不能作为全局对象的属性来访问
  • const定义常量，定义后不能再被修改
  • const声明时必须赋值
